# トリモドセ 設計書

## 概要

情報の非対称性を破壊する「経済的武装」プラットフォーム。
若者が知らないだけで損している給与相場との差額、未申請の控除・給付金を可視化し、具体的な取り戻し方を提示する。

「夢を持たせる」のではなく「お前は奪われている」と気づかせるツール。

## ターゲット

- **MVP**: 新卒1-3年目の正社員
- **将来展開**: フリーランス・副業勢 → 非正規雇用（派遣・バイト）

## UXフロー

### Step 1: 概算診断（10秒以内）

ユーザーが3項目を入力する:
- 年収（万円）
- 職種（選択式）
- 地域（都道府県）

即座に「あなたの推定損失額: 年間XX万円」を表示。公開統計データ（厚労省 賃金構造基本統計調査、国税庁 民間給与実態統計）との比較。

### Step 2: 精密診断

概算で怒りに火がついたユーザーに、Yes/No質問を10-12問提示。1問ずつウィザード形式で表示し、離脱を防ぐ。

| # | 質問 | 関連控除 |
|---|------|---------|
| 1 | 年間の医療費は10万円を超えましたか？ | 医療費控除 |
| 2 | ふるさと納税をしていますか？ | 寄附金控除 |
| 3 | iDeCo（個人型確定拠出年金）に加入していますか？ | 小規模企業共済等掛金控除 |
| 4 | 生命保険に加入していますか？ | 生命保険料控除 |
| 5 | 地震保険に加入していますか？ | 地震保険料控除 |
| 6 | 住宅ローンを組んでいますか？ | 住宅借入金等特別控除 |
| 7 | 配偶者はいますか？（年収制限あり） | 配偶者控除/配偶者特別控除 |
| 8 | 扶養している家族はいますか？ | 扶養控除 |
| 9 | 働きながら学校に通っていますか？ | 勤労学生控除 |
| 10 | 災害・盗難の被害を受けましたか？ | 雑損控除 |
| 11 | 年末調整で申告し忘れた控除はありますか？ | 確定申告による還付 |
| 12 | 通勤手当は非課税枠を超えていませんか？ | 通勤費の非課税限度額 |

Yesの場合のみ追加入力を求める（例: 医療費Yesなら「いくらでしたか？」）。

### Step 3: 結果表示

該当する控除のリスト、各控除の節約額、合計取り戻し額、根拠条文（所得税法の条文番号）を表示。

### Step 4: アカウント作成（任意）

結果を見た後に「結果を保存する」で匿名アカウント作成に誘導。価値を感じてからの登録。

## UIデザイン方針

クリーン&シリアス。白背景、赤は損失額だけに使用。金融サービス的な信頼感。数字の説得力重視。

- 白背景ベース
- 赤（#dc2626）は損失額・警告のみ
- shadcn/uiのデフォルトテーマをベースに
- スマホファーストのレスポンシブデザイン

## ルールエンジン

控除ルールはDBの`deduction_rules`テーブルにデータとして保持。法改正時はコード変更不要、データ更新のみで対応。

処理フロー:
1. ユーザーのYes/No回答を受け取る
2. `deduction_rules`テーブルを走査
3. 各控除について`conditions`を照合 → 該当判定
4. 該当する場合`formula`を適用 → 節約額を計算
5. `legal_basis`から根拠条文を紐付け
6. 結果を返す: `[{ name, saving, legalBasis, howTo }]`

ルール定義の例:
```json
{
  "question_key": "medical_over_100k",
  "condition": "answer === true",
  "formula": "(medical_amount - 100000) * tax_rate",
  "cap": null,
  "legal_basis": "所得税法第73条"
}
```

「やっていない」が該当する場合を重視する。「iDeCoやってない → 年間XX万の節税機会を逃してる」が核心の価値。

## テスト戦略

テストフレームワーク: Vitest

### 優先度

| 優先度 | 対象 | 理由 |
|--------|------|------|
| 最高 | ルールエンジン (deduction-engine) | 間違った控除額の表示は信頼喪失に直結 |
| 最高 | 計算ロジック (calculator) | 税率計算、控除額計算の精度は命 |
| 高 | 統計比較 (salary-comparator) | 年齢・職種の照合ミスは損失額の誤表示に直結 |
| 通常 | API | 入出力の型チェック |
| 通常 | UI | ウィザード遷移、フォームバリデーション |

### テストケース例

ルールエンジンは法令ベースのテストケースを書く:
- 年収400万、医療費15万 → 医療費控除 = (150,000 - 100,000) x 20% = 10,000円
- iDeCo月額23,000円 → 小規模企業共済等掛金控除 = 276,000円 x 税率

## MVPスコープ

### 入れる

- 概算診断（3項目入力 → 損失額表示）
- 精密診断（Yes/No質問 → 控除リスト → 取り戻し額）
- 根拠条文表示
- 匿名アカウント（メールのみ）
- 結果保存（暗号化）
- レスポンシブ（スマホファースト）

### 入れない

- フリーランス/非正規対応
- 給与明細OCR
- AI診断
- SNSシェア
- 過去診断との比較
- 多言語対応
- ふるさと納税シミュレータ

## 認証・プライバシー

- メールアドレスのみで登録（名前不要）
- 診断データ（`diagnoses.input`）はアプリケーションレベルで暗号化して保存
- 未ログインでも診断可能（`user_id` nullable）
- アカウント作成は結果表示後に誘導
